<size=16>AnyPortrait Update Log v1.6.2</size>
(October 7, 2025)

<size=13>Improvements</size>

- <color=blue>Improved editor usability</color>
    Due to a conflict between Unity Editor shortcuts and the Gizmo change shortcuts (default Q, W, E, R), these shortcuts have been fixed to no longer work when special keys like Ctrl are pressed.
    The UI colors for non-changeable shortcuts in the Shortcut Settings dialog have been changed to better distinguish them in the Dark theme.
    A UI has been added that allows numeric input of the mesh's pivot.
    A button has been added to move the mesh pivot to the center of the vertices.
    When modifying vertex positions in the Modify tab of a mesh, UVs are now automatically updated, and user edits are no longer possible.
    Links to tutorials and contact pages in the Unity menu have been consolidated into a single "Tutorials and Support" menu.

- The texture type of script functions that change custom properties, such as "SetMeshCustomImageAll," has been changed from Texture2D to Texture, allowing inputs such as RenderTexture.
- Due to the separation of the Unity Asset Store, packages are rebuilt depending on the store they are uploaded to.



<size=13>Fixed issues</size>

- Thanks to active user participation, many Chinese translation errors have been corrected.
- An issue has been fixed where an error would occur in a script function that batch-controlled materials when all meshes of a specific material were included in the mask system.
- An issue has been fixed where compatibility warnings were occurred due to script API changes in Unity 6.2.
